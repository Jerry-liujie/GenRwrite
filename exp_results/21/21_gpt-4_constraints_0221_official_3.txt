1. Use CTEs to break down complex queries: This rule can be applied when the query involves multiple stages of data processing, such as aggregations or transformations, that can be separated into distinct steps.
2. Move conditions to the earliest possible point in the query: This rule can be applied when there are conditions in the WHERE or JOIN clauses that can be moved to an earlier point in the query to reduce the amount of data processed in later stages.
3. Use JOIN instead of comma-separated FROM clause: This rule can be applied when the query involves joining multiple tables, and the join conditions can be specified in a JOIN clause instead of a comma-separated list in the FROM clause.
