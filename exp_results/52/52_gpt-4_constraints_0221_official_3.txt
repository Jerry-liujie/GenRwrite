1. Use explicit JOIN syntax instead of comma-separated FROM clause: This rule can be applied when multiple tables are being joined in a query. It helps in clearly specifying the type of join and the joining condition.
2. Move conditions from WHERE clause to ON clause in JOIN: This rule can be applied when there are conditions in the WHERE clause that are based on columns from the joined tables. Moving these conditions to the ON clause can improve performance.
3. Use indexes on columns used in JOIN, WHERE, GROUP BY and ORDER BY clauses: This rule can be applied when a query involves operations on large datasets and the columns used in these operations are not indexed. Creating indexes on these columns can significantly improve the query performance.
