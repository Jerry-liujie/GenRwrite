1. Filter data in subqueries: This rule can be applied when the query involves joining multiple tables and there are conditions that can be used to filter the data in the individual tables before the join operation.
2. Remove unnecessary join: This rule can be applied when a table is joined in the query but none of its columns are used in the select, where, group by, or order by clauses.
3. Use explicit join syntax: This rule can be applied to any query that uses implicit join syntax. It is a best practice to always use explicit join syntax for clarity and readability.
