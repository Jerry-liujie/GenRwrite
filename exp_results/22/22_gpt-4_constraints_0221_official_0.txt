1. Remove unnecessary joins: This rule can be applied when there are tables joined in the query that do not contribute to the final result set, either by not providing any columns in the SELECT clause or not being part of any WHERE, GROUP BY, or HAVING conditions.
2. Use subquery to reduce the amount of data: This rule can be applied when a large table is joined with another table, and only a small subset of the large table's data is needed for the final result. A subquery can be used to filter the large table down to the necessary rows before the join operation.
3. Group by before join: This rule can be applied when a table is being grouped and then joined with another table. By grouping the data before the join operation, the amount of data that needs to be processed during the join can be reduced, which can improve performance.
