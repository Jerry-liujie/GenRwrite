1. Push down predicates: This rule can be applied when there are filtering conditions in the query that can be moved closer to the data source to reduce the amount of data that needs to be processed in the subsequent stages of the query.
2. Use subquery to pre-calculate case statements: This rule can be applied when there are case statements in the query that can be moved into a subquery and converted to integers to make the sum function in the outer query more efficient.
3. Replace join with in clause for filtering: This rule can be applied when a join operation is used for filtering purposes in the query, and can be replaced with an in clause to avoid the computational cost of the join operation.
