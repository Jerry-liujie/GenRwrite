1. Use Common Table Expressions (CTEs) to simplify the query: This rule can be applied when the query involves multiple complex operations that can be broken down into simpler, individual queries.
2. Filter data as early as possible: This rule can be applied when there are conditions that can be used to filter the data in the early stages of the query.
3. Avoid unnecessary joins: This rule can be applied when there are joins in the query that do not contribute to the final result set.
