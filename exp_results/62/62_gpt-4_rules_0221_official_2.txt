1. Push down predicates: This rule pushes the filtering conditions as close to the data source as possible. This reduces the amount of data that needs to be processed in the subsequent stages of the query. In this case, the condition on `d_month_seq` is pushed down to the subquery.
2. Use subquery to pre-calculate case statements: This rule moves the case statements into a subquery and converts the boolean results to integers. This allows the sum function in the outer query to simply add up the integers, which is more efficient than evaluating the case statements for each row.
3. Replace join with in clause for filtering: This rule replaces a join operation used for filtering with an in clause. This can be more efficient because it avoids the need to perform a join operation, which can be expensive in terms of computational resources.
