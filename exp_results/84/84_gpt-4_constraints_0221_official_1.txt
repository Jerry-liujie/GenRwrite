1. Use explicit JOIN syntax instead of comma-separated tables in the FROM clause: This rule can be applied when the query involves joining multiple tables and the join conditions are specified in the WHERE clause.
2. Move conditions from the WHERE clause to the ON clause in JOINs: This rule can be applied when there are conditions in the WHERE clause that are related to the joined tables.
3. Use table aliases for better readability: This rule can be applied when the query involves multiple tables, and using the full table names in the query makes it hard to read and understand.
4. Filter data as early as possible: This rule can be applied when there are conditions that can be used to filter the data in the early stages of the query.
