1. Use explicit JOIN syntax instead of comma-separated FROM clause: This rule can be applied when multiple tables are being joined in a query, especially when there are complex join conditions.
2. Move the WHERE clause filters to the subquery: This rule can be applied when there are filtering conditions that can be moved to a subquery to reduce the amount of data that needs to be processed in the main query.
3. Remove unnecessary columns from the GROUP BY clause: This rule can be applied when there are columns in the GROUP BY clause that are not included in the SELECT clause or used in an aggregate function.
