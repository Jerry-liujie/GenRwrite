1. Use explicit JOIN syntax instead of comma-separated tables in the FROM clause: This rule can be applied when multiple tables are being used in a query and there is a need to specify the relationship between these tables.
2. Use table aliases for tables with long names: This rule can be applied when the query involves tables with long or complex names, or when the same table is used multiple times in the query.
3. Only select necessary columns: This rule can be applied when the query is selecting more columns than necessary, especially when these extra columns are not used in the WHERE, GROUP BY, or ORDER BY clauses.
