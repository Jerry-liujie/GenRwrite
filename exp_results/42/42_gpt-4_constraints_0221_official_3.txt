1. Replace implicit join with explicit join: This rule can be applied when the query uses implicit join syntax (comma-separated table list in the FROM clause and join conditions in the WHERE clause).
2. Use table aliases for better readability: This rule can be applied to any query that involves more than one table. It is especially useful when the same table is used multiple times in the query or when the table names are long or complex.
3. Filter in the WHERE clause: This rule can be applied when there are conditions that can be used to filter the data before performing operations like grouping or aggregating.
