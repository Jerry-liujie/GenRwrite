1. Use explicit JOIN syntax instead of comma-separated FROM clause: This rule can be applied when multiple tables are being joined in a query, especially when there are complex join conditions.
2. Use CTEs (Common Table Expressions) instead of subqueries: This rule can be applied when a subquery is used multiple times in the main query or when the subquery is complex and makes the main query difficult to read.
3. Remove unnecessary parentheses: This rule can be applied when there are parentheses in the query that do not contribute to the logical order of operations and only serve to make the query more complex and difficult to read.
