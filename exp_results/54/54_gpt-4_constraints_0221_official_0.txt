1. Use explicit JOIN syntax instead of implicit: This rule can be applied when the query uses implicit JOIN syntax (i.e., listing tables in the FROM clause and specifying join conditions in the WHERE clause).
2. Split UNION ALL subquery into two separate subqueries: This rule can be applied when a UNION ALL operation is used to combine results from two similar subqueries that differ only in the table they select from.
3. Remove unnecessary DISTINCT keyword: This rule can be applied when the DISTINCT keyword is used in a query, but the data is already unique or the duplicates do not affect the final result.
4. Remove unnecessary columns from the SELECT clause: This rule can be applied when the SELECT clause includes columns that are not used in the final result or in subsequent stages of the query.
