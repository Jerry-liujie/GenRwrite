1. Replace implicit joins with explicit joins: This rule can be applied when the query uses a comma-separated list of tables in the FROM clause and specifies the join conditions in the WHERE clause.
2. Use "select 1" in subqueries: This rule can be applied when a subquery is used in the WHERE clause to check for the existence of rows, and the actual data returned by the subquery is not used.
3. Move conditions from WHERE clause to ON clause in JOINs: This rule can be applied when there are conditions in the WHERE clause that involve columns from more than one table.
4. Use BETWEEN for range conditions: This rule can be applied when there are conditions in the WHERE clause that check if a column's value is greater than or equal to a certain value and less than or equal to another value.
