1. Replace implicit joins with explicit joins: This rule can be applied when the query uses implicit joins, which are represented by commas in the FROM clause and conditions in the WHERE clause.
2. Use SELECT 1 instead of SELECT * in subqueries: This rule can be applied when the subquery is used for existence check only, and the actual data returned by the subquery is not used.
3. Move the NOT EXISTS conditions inside the EXISTS subquery: This rule can be applied when the query has multiple EXISTS and NOT EXISTS subqueries that are based on the same main query.
