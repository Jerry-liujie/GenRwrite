1. Use JOIN instead of WHERE for combining tables: This rule can be applied when multiple tables are being combined based on matching column values, which is typically done using WHERE clause but can be more efficiently done using JOIN.
2. Push conditions into JOIN ON clause: This rule can be applied when there are conditions in the WHERE clause that are based on columns from two tables that are being joined. These conditions can be moved to the ON clause of the JOIN for better performance and readability.
3. Remove unnecessary table aliases: This rule can be applied when tables in the query have been given aliases but these aliases are not required for disambiguation or for making the query more concise.
