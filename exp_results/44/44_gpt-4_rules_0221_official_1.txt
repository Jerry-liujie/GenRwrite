1. Use CTEs (Common Table Expressions) instead of subqueries: CTEs improve readability and maintainability of complex queries. They also allow you to break down complex queries into simpler parts.
2. Combine multiple subqueries into a single subquery where possible: This reduces the number of scans needed on the same table, improving performance.
3. Use JOIN instead of WHERE for linking tables: JOINs are generally faster and more efficient than WHERE clauses for combining rows from two or more tables.
4. Move the condition inside the window function to the WHERE clause of the main query: This can improve performance by reducing the number of rows that need to be processed by the window function.
