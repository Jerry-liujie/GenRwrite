1. Use WITH clause to simplify the query: This rule can be applied when the query involves multiple subqueries or complex operations that can be isolated for better readability and maintainability.
2. Combine subqueries into one using JOINs: This rule can be applied when there are multiple subqueries that operate on the same tables and can be combined into a single query.
3. Move the WHERE conditions to the WITH clause: This rule can be applied when there are conditions in the main query that can be moved to the subquery to reduce the number of records processed in the main query.
4. Remove unnecessary nested subqueries: This rule can be applied when there are nested subqueries that can be simplified or eliminated without changing the result of the query.
