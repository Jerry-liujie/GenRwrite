1. Use explicit JOIN syntax instead of comma-separated tables in the FROM clause: This rule can be applied when multiple tables are being joined in a query, especially when there are complex join conditions.
2. Combine multiple AND conditions in WHERE clause: This rule can be applied when there are multiple conditions in the WHERE clause that can be combined using AND operator.
3. Remove unnecessary conditions in WHERE clause: This rule can be applied when there are conditions in the WHERE clause that do not contribute to the result of the query or are redundant.
4. Remove duplicated conditions: This rule can be applied when there are duplicated conditions in the WHERE clause or in the JOIN conditions.
5. Use table aliases for better readability: This rule can be applied when the query involves multiple tables, especially when the table names are long or complex.
