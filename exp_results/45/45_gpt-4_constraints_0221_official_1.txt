1. Use WITH clause for subqueries: This rule can be applied when the query involves subqueries that are complex or used multiple times in the main query.
2. Use explicit JOIN syntax instead of comma-separated tables: This rule can be applied when the query involves joining multiple tables. Using explicit JOIN syntax can make the query more readable and easier to understand.
3. Move conditions to the ON clause of the JOIN statement: This rule can be applied when there are conditions that can be used to filter the data during the join operation. This can improve performance by reducing the number of rows that need to be joined.
