1. Use WITH clause for subqueries: This rule can be applied when the query contains subqueries that are used multiple times or when the subquery is complex and makes the main query difficult to read.
2. Replace implicit JOINs with explicit JOINs: This rule can be applied when the query uses implicit JOIN syntax (comma-separated table list in the FROM clause and join conditions in the WHERE clause).
3. Pre-calculate aggregates: This rule can be applied when the query contains aggregate functions that are calculated multiple times or when the aggregate calculation is complex.
4. Use subquery to filter rows: This rule can be applied when the query contains conditions that can be used to filter the data before the main query is executed. This is especially useful when the filtered data significantly reduces the number of rows processed in the main query.
