1. Remove unnecessary joins: This rule can be applied when there are joins in the query that do not contribute to the final result. This can be identified by checking if the columns from the joined tables are not used in the SELECT, WHERE, GROUP BY or ORDER BY clauses.
2. Use IN clause for subquery: This rule can be applied when there is a subquery that is used to filter the data based on a range of values. This can be identified by checking if the WHERE clause of the main query uses a BETWEEN operator to filter the data based on the result of a subquery.
3. Use explicit join for catalog sales and returns: This rule can be applied when there are cross joins in the query that can be replaced with explicit joins. This can be identified by checking if the FROM clause of the query includes a comma-separated list of tables without a corresponding WHERE clause that specifies the join condition.
