1. Use join instead of subquery in where clause: This rule can be applied when there are subqueries in the WHERE clause that can be replaced with JOINs to improve performance.
2. Filter data as early as possible: This rule can be applied when there are conditions that can be used to filter the data in the early stages of the query.
3. Avoid repeated subqueries: This rule can be applied when the same subquery is used multiple times in the query, which can be replaced with a single subquery to improve performance.
4. Use union all instead of union: This rule can be applied when the query is combining results from multiple SELECT statements and it is not necessary to remove duplicate rows.
