3. Combine conditions on the same column with IN instead of multiple ORs: This rule can be applied when there are multiple OR conditions on the same column in the WHERE clause.
4. Use GROUP BY and HAVING instead of multiple self-joins: This rule can be applied when the query contains multiple self-joins that are used to perform calculations on groups of rows.
5. Use CASE in the HAVING clause to perform calculations on specific rows: This rule can be applied when there are calculations that need to be performed on specific rows based on certain conditions.
