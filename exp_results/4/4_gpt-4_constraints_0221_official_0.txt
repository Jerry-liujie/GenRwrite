1. Filter early and often: This rule can be applied when there are conditions that can significantly reduce the amount of data that needs to be processed in the later stages of the query.
2. Use explicit JOINs instead of comma-separated FROM clauses: This rule can be applied when there are multiple tables in the FROM clause that are joined based on certain conditions.
3. Avoid unnecessary calculations: This rule can be applied when the same calculations are performed multiple times in the query. By calculating the values once and then referencing them later, we can potentially improve performance.
