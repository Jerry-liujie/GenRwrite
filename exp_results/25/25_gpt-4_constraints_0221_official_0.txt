1. Aggregate before join: This rule can be applied when the query involves joining tables on keys that have been grouped and aggregated, as this can reduce the size of the tables before the join operation.
2. Push down predicates: This rule can be applied when there are filtering conditions that can be moved closer to the table scan operation, which can reduce the amount of data that needs to be processed.
3. Use subqueries to break down complex queries: This rule can be applied when the query is complex and involves multiple operations, as breaking down the query into subqueries can make it more readable and maintainable, and can also improve performance.
