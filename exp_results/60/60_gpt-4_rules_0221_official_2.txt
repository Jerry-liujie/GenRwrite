1. Combine multiple subqueries into one: This reduces the number of scans on the same table and reduces the number of subqueries. It improves performance by reducing the amount of data to be processed.
2. Use LEFT JOIN instead of IN clause: The IN clause can be inefficient when dealing with large data sets. Using a LEFT JOIN can be more efficient as it can take advantage of indexes.
3. Use CASE statement to sum sales from different tables: This avoids the need to union the results from different tables, which can be costly in terms of performance.
4. Move conditions from WHERE clause to ON clause in LEFT JOIN: This can improve performance by reducing the amount of data that needs to be joined.
