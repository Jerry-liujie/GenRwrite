1. Use CTEs for complex calculations: This rule can be applied when the query contains complex calculations that can be encapsulated in a Common Table Expression to improve readability and potentially performance. This is beneficial as it simplifies the main query and makes it easier to understand.
2. Move conditions from the outer query to the join condition: This rule can be applied when there are conditions in the WHERE clause that involve columns from both tables in the join. This is beneficial as it can reduce the amount of data that needs to be processed in the join operation, thereby improving performance.
3. Use a case statement in the sum function: This rule can be applied when there are conditional calculations within the aggregate function. This is beneficial as it simplifies the query and can potentially improve performance by reducing the number of calculations needed.
