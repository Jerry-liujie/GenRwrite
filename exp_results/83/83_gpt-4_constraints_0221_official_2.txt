1. Use JOIN instead of WHERE for combining tables: This rule can be applied when multiple tables are being combined based on a common column, and the result is being filtered based on conditions related to these tables.
2. Combine conditions in JOIN ON clause: This rule can be applied when there are conditions that can be used to filter the data while joining tables, instead of filtering the data after the tables have been joined.
3. Avoid subqueries in WHERE clause: This rule can be applied when there are subqueries in the WHERE clause that can be moved to the ON clause in a JOIN, which can improve performance by reducing the amount of data that needs to be processed.
