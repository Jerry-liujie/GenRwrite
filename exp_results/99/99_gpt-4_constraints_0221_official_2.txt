1. Use explicit JOIN syntax instead of comma-separated FROM clause: This rule can be applied when multiple tables are being joined in a query, especially when there are complex join conditions.
2. Move conditions from WHERE clause to ON clause in JOINs: This rule can be applied when there are conditions in the WHERE clause that are related to the joined tables and can be moved to the ON clause to filter data earlier.
3. Replace SUM with COUNT for conditional counting: This rule can be applied when the query is using SUM function to count the number of rows that meet a certain condition, as COUNT function can achieve the same result more efficiently.
