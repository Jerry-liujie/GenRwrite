1. Use JOIN instead of comma-separated tables in FROM clause: This rule can be applied when multiple tables are being used in the query and there is a relationship between them that can be expressed using a JOIN operation.
2. Avoid using DISTINCT when not necessary: This rule can be applied when the query uses set operations like UNION, INTERSECT, or EXCEPT, which automatically remove duplicate rows.
3. Filter data as early as possible: This rule can be applied when there are conditions that can be used to filter the data in the early stages of the query, reducing the amount of data that needs to be processed in later stages.
