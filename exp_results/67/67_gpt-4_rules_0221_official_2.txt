1. Use JOIN instead of WHERE for table relationships: This makes the query more readable and easier to understand. It also allows for better optimization by the database engine.
2. Remove ROLLUP function: The ROLLUP function generates a result set that is similar to the one generated by the GROUP BY clause, but with additional rows that represent subtotals and grand totals. However, in this case, we don't need these additional rows, so removing the ROLLUP function can improve performance.
3. Filter in subquery: By filtering the data in the subquery, we reduce the amount of data that needs to be processed in the outer query, which can improve performance.
