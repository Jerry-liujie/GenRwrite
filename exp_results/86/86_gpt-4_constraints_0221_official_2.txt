1. Use CTE (Common Table Expressions) to simplify complex joins: This rule can be applied when the query involves multiple joins or subqueries, which can be simplified and made more readable by using CTEs.
2. Filter data as early as possible: This rule can be applied when there are conditions that can be used to filter the data in the early stages of the query, reducing the amount of data that needs to be processed in later stages.
3. Avoid using SELECT *: This rule can be applied when the query does not require all columns from the tables being queried. Instead, only the necessary columns should be specified in the SELECT clause.
