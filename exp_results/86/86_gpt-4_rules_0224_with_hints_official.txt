1. Use CTE (Common Table Expressions) to break down complex queries: This rule improves readability and maintainability of the code by breaking down the complex query into simpler parts. It also improves performance by avoiding repeated computation of the same subquery.
2. Replace implicit joins with explicit joins: This rule improves readability and maintainability of the code by making the join operations explicit. It also improves performance by allowing the database engine to optimize the join operations.
3. Move the filtering condition to the end: This rule improves performance by reducing the amount of data to be processed. By applying the filtering condition after the join operation, only the relevant data is processed in the subsequent stages of the query.
4. Use join instead of comma-separated tables in the from clause: This rule improves readability and performance by making the join operations explicit and allowing the database engine to optimize the join operations.
