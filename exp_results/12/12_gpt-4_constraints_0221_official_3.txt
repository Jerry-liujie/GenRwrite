1. Subquery to limit rows: This rule can be applied when there are conditions that can filter a large table before it is joined with other tables, reducing the number of rows that need to be processed.
2. Use IN instead of JOIN for filtering: This rule can be applied when the purpose of the join is to filter rows based on a single column from one table that matches a single column in another table.
3. Push down predicates: This rule can be applied when there are conditions on columns from the joined tables that can be used to filter out rows before the join operation, reducing the amount of data that needs to be processed.
