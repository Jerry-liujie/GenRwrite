1. Use JOIN instead of WHERE for table relationships: This rule can be applied when there are multiple tables in the query that are related to each other through certain columns.
2. Pre-calculate averages in a subquery: This rule can be applied when the query involves calculating averages or other aggregate functions that can be pre-calculated in a subquery to simplify the main query.
3. Use CTEs (Common Table Expressions) for complex subqueries: This rule can be applied when the query is complex and can be broken down into simpler parts, each of which can be represented as a CTE.
