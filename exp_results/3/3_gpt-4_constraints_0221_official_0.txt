1. Use explicit JOIN syntax instead of comma-separated tables in the FROM clause: This rule can be applied when multiple tables are being queried together, and there are relationships between these tables that can be expressed as JOIN operations.
2. Move conditions from WHERE clause to ON clause in JOINs where possible: This rule can be applied when there are conditions in the WHERE clause that involve columns from multiple tables, and these conditions can be moved to the ON clause of the relevant JOIN operation.
3. Use LIMIT clause to reduce the number of rows returned: This rule can be applied when the query is expected to return a large number of rows, but only a small subset of these rows is actually needed for the analysis.
