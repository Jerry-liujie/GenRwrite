1. Replace implicit joins with explicit joins: This rule can be applied when the query uses implicit joins, which are represented by commas between table names in the FROM clause and conditions in the WHERE clause.
2. Filter in the WHERE clause before joining tables: This rule can be applied when there are conditions that can be used to filter the data before the join operation. This is typically applicable when the filter conditions are based on columns from one table only.
3. Use LIMIT clause to reduce the amount of data returned: This rule can be applied when the client doesn't need to see all the data, but only a subset of it, typically the top or bottom records based on some order.
