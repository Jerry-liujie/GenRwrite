2. Use JOIN instead of comma-separated FROM clause: This rule can be applied when the query involves multiple tables and there are conditions that can be used to join the tables, improving the readability and efficiency of the query.
3. Avoid cross join: This rule can be applied when the query involves multiple tables and there are no explicit join conditions, resulting in a cross join. Adding explicit join conditions can avoid unnecessary computation and improve performance.
