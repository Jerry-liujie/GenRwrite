1. Subquery for filtering: Instead of joining all tables and then applying the filters, we first filter the data in each table using subqueries and then join the results. This can significantly reduce the amount of data that needs to be processed and joined, thus improving performance.
2. Use IN clause for filtering: Instead of joining tables to filter data, we use the IN clause. This can be faster if the subquery returns a small number of rows.
3. Avoid unnecessary columns in the select clause: Only select the columns that are necessary for the final result. This reduces the amount of data that needs to be processed and returned, thus improving performance.
