1. Replace joins with subqueries: This rule can be applied when the join operation is used for filtering data based on certain conditions, and the number of records returned by the subquery is significantly less than the total number of records in the joined table.
2. Filter data as early as possible: This rule can be applied when there are conditions that can be used to filter the data in the early stages of the query, especially when the dataset is large.
3. Use IN clause for filtering: This rule can be applied when there are multiple OR conditions in the WHERE clause that check if a column's value is among a list of values.
