1. Use subqueries to filter data: This rule can be applied when there are specific conditions that can be used to filter the data before it is processed in the main query.
2. Use IN clause instead of JOIN for filtering: This rule can be applied when the number of records that meet the conditions is small and the data needs to be filtered based on specific values.
3. Join only necessary tables in the main query: This rule can be applied when the query involves multiple tables but not all of them are needed to get the final result.
4. Filter data as early as possible: This rule can be applied when there are conditions that can be used to significantly reduce the amount of data that needs to be processed in the later stages of the query.
