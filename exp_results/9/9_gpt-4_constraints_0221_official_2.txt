1. Use of CTE (Common Table Expressions): This rule can be applied when there are multiple subqueries that are querying the same data or when the query is complex and can be broken down into smaller, simpler queries.
2. Avoiding multiple subqueries: This rule can be applied when the same subquery is used multiple times in the main query. Instead of repeating the subquery, it can be replaced with a CTE.
3. Pre-aggregation: This rule can be applied when there are aggregate functions used in the main query. By pre-aggregating the data in a CTE, the amount of data that needs to be processed in the main query can be reduced.
