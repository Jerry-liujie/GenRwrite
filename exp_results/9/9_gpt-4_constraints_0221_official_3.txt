1. Use of Common Table Expressions (CTEs): This rule can be applied when there are complex subqueries that are used multiple times in the main query. By using CTEs, we can avoid repeated execution of the same subquery.
2. Avoiding subqueries in SELECT clause: This rule can be applied when there are subqueries in the SELECT clause, which are executed for each row returned by the main query. By moving these subqueries to a CTE, they are only executed once, improving performance.
3. Combining count and average calculations in a single query: This rule can be applied when there are multiple aggregate functions like COUNT, AVG, SUM, etc. applied on the same column or set of columns. By combining these calculations into a single query, we can reduce the number of times the database needs to scan the table.
