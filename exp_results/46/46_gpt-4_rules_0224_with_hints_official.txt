1. Use alias for tables: This makes the query more readable and easier to maintain.
2. Use explicit JOIN syntax instead of comma-separated tables in FROM clause: This makes the relationships between the tables clearer and the query easier to understand.
3. Grouping in subquery: This reduces the amount of data that needs to be processed in the outer query and improves performance.
4. Remove redundant values from IN clause: This improves the performance of the query.
5. Move conditions from WHERE clause to ON clause in JOINs where possible: This can improve performance by reducing the amount of data that needs to be processed.
