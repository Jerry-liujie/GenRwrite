1. Use explicit JOIN syntax instead of comma-separated FROM clause: This rule can be applied when multiple tables are being joined in a query. Using explicit JOIN syntax can make the query easier to understand and often leads to better performance.
2. Move conditions into subquery: This rule can be applied when there are conditions that can significantly reduce the number of rows that need to be joined. Moving these conditions into a subquery can improve performance by reducing the amount of data that needs to be processed.
3. Use subquery to pre-filter rows: This rule can be applied when there are conditions that can be used to filter out rows before they are processed in the main query. Using a subquery to pre-filter rows can improve performance by reducing the amount of data that needs to be processed.
4. Remove unnecessary columns from GROUP BY clause: This rule can be applied when there are columns in the GROUP BY clause that are not included in the SELECT clause or used in an aggregate function. Removing these unnecessary columns can improve performance by reducing the amount of data that needs to be grouped.
