1. Push down predicates: This rule can be applied when there are conditions in the main query that can be moved to a subquery to filter data at an earlier stage, reducing the amount of data that needs to be processed later.
2. Use subqueries to reduce data: This rule can be applied when there are conditions on specific tables that can be used to filter data. By creating a subquery for these tables, the amount of data that needs to be joined in the main query can be reduced.
3. Remove unnecessary cast operations: This rule can be applied when the data types of the columns and the constants used in the conditions match, and there are unnecessary cast operations in the query that can be removed to reduce computation and improve performance.
