1. Move conditions into subquery: This rule can be applied when there are conditions in the main query that can be moved into a subquery to reduce the amount of data that needs to be joined.
2. Remove unnecessary cast operations: This rule can be applied when there are cast operations in the query that are not necessary for the computation or the result.
3. Push down predicates: This rule can be applied when there are conditions in the main query that can be moved to a lower level (closer to the data source) in the query to reduce the amount of data that needs to be processed in the later stages.
