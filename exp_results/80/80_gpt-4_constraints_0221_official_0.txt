1. Remove unnecessary subqueries: This rule can be applied when the subqueries in the main query are not dependent on the data from the main query or other subqueries and can be executed independently.
2. Use UNION ALL instead of UNION: This rule can be applied when the result sets from your queries do not overlap, and you are sure that there will be no duplicate rows.
3. Remove the ROLLUP function: This rule can be applied when subtotals and grand totals generated by the ROLLUP function are not required in the final output.
4. Limit the number of rows returned: This rule can be applied when only a specific number of rows are required for the output, not the entire result set.
5. Use indexing: This rule can be applied when the columns used in the WHERE, JOIN and ORDER BY clauses are frequently accessed and these operations are slowing down the query performance.
