2. Replace IN subquery with JOIN: This can improve performance because the database can use a hash join or merge join instead of a nested loops join.

4. Move the ranking condition to a HAVING clause in the subquery: This can improve performance by reducing the number of rows returned by the subquery.
5. Use column aliases for calculated columns: This makes the query more readable and allows the calculated columns to be referenced elsewhere in the query.
