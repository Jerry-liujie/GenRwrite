1. Use explicit JOIN syntax instead of comma-separated tables in FROM clause: This rule can be applied when multiple tables are being queried and there are relationships between these tables that can be expressed as joins.
2. Use IN clause instead of multiple AND conditions for matching multiple columns: This rule can be applied when there are conditions that involve matching multiple columns from different tables.
3. Avoid subqueries in the SELECT clause: This rule can be applied when there are subqueries in the SELECT clause that can be rewritten as joins.
4. Use WHERE clause to filter rows early: This rule can be applied when there are conditions that can be used to filter the data in the early stages of the query.
