3. Combine similar subqueries: This rule can be applied when the query contains multiple subqueries that perform similar operations on the same or related data.
4. Filter as early as possible: This rule can be applied when the query involves large amounts of data and there are conditions that can be used to reduce the amount of data that needs to be processed.
5. Use table aliases to shorten table names: This rule can be applied when the query involves tables with long or complex names, or when the same table is used multiple times in the query.
