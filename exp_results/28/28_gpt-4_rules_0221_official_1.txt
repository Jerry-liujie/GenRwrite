1. Avoid using subqueries in FROM clause: Subqueries in FROM clause create derived tables for each subquery which can be expensive in terms of memory and processing time. Instead, we can use subqueries in the SELECT clause to calculate the required values. This can improve the performance by reducing the number of derived tables created and processed.
2. Use LIMIT clause wisely: The LIMIT clause can be used to restrict the number of rows returned by the query. However, it should be used wisely as it can lead to unexpected results if not used correctly. In this case, the LIMIT clause is not necessary as the subqueries in the SELECT clause will return only one row each.
