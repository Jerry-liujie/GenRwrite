1. Use of Common Table Expressions (CTEs): This rule is applied to simplify the query by breaking it down into smaller, more manageable pieces. This makes the query easier to read and understand, and can also improve performance by allowing the database to cache the results of the CTEs for reuse in the main query.
2. Consolidate where clause conditions: This rule is applied to simplify the WHERE clause by combining multiple conditions using logical operators (AND, OR). This can make the query easier to read and understand, and can also improve performance by reducing the number of conditions the database needs to evaluate.
3. Avoiding subqueries in SELECT clause: This rule is applied to improve performance by reducing the number of subqueries that the database needs to execute. Instead of executing a subquery for each row returned by the main query, the database can execute the subquery once and reuse the results.
